# 要件定義サマリー
## プロジェクト名: Ardors

### 1. エグゼクティブサマリー
Ardorsを「開発スタジオ（受託・技術コンサル）」として伝えるコーポレートサイトの要件定義を完了した。  
主要目的は、問い合わせ獲得、実績/プロフィールの構造的提示、SaaS構想の導線確立である。  
公開サイトはログイン不要とし、P0では情報訴求・問い合わせ・セキュリティ・UX品質を重視する。  
P1で事例紹介CMS、計測強化、法務拡張を段階導入する。  
Phase 6で未確定項目を全件確定し、実装に移行できる状態にした。

### 3. 主要決定事項（要約）
- サイト種別: Ardorsコーポレートサイト（受託・技術コンサル訴求 + SaaS構想導線）。
- 主要ターゲット: 見込み顧客、知人/取引先、協業希望者、志望先企業担当者。
- 公開サイト方針: 閲覧側ログイン不要、日本語/英語対応。
- 外部連携: Note RSS、GA4、Search Console、GitHub導線。
- 事例紹介CMS: 初期P1、テキスト中心、画像添付なし、管理操作はU-05限定。
- セキュリティ基準: サーバー側入力検証、レート制限（1分5件/IP）、セキュアヘッダー、CVSS 7.0以上 0件。
- 変更容易性方針: 共通UI分離、単一ナビ定義、画面追加時3点更新（画面一覧/遷移図/関連機能ID）。
- 旧SaaS系資料: 現行要件との混在を防ぐため削除方針で確定。
