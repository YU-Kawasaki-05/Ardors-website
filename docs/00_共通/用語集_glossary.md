# 用語集

| 用語 | 定義 | 補足 | 登場ドキュメント |
|------|------|------|----------------|
| Ardors | 本プロジェクトの屋号（将来的に社名化の可能性あり） | 開発スタジオブランド | 01_業務理解 |
| コーポレートサイト | 企業/事業の情報を統合して伝える公式Webサイト | 信頼形成と問い合わせ導線の中核 | 01_業務理解 |
| ポートフォリオ | 実績・スキル・プロフィールを示す公開情報 | 受託/協業の判断材料 | 01_業務理解 |
| SaaS構想 | 将来提供予定の自社プロダクト計画 | 初期は紹介と導線中心 | 01_業務理解 |
| CTA | ユーザーに行動を促す導線（例: 問い合わせ） | Conversion最適化に重要 | 01_業務理解 |
| GA4 | Google Analytics 4 | サイト行動計測に利用 | 01_業務理解 |
| Search Console | Google Search Console | 検索パフォーマンス把握に利用 | 01_業務理解 |
| 見込み顧客 | サービス導入や相談を検討している訪問者 | 主要ターゲット | 02_ユーザー像 |
| 協業希望者 | Ardorsと共同で仕事を進めたい訪問者 | パートナー候補 | 02_ユーザー像 |
| 志望先企業担当者 | 就職選考文脈で能力評価を行う担当者 | 職務適合性を確認 | 02_ユーザー像 |
| 権限マトリクス | ユーザー種別ごとの操作可否を整理した表 | 認可要件の基礎資料 | 02_ユーザー像 |
| Core Web Vitals | Web UX品質の主要指標群 | LCP/INP/CLSを含む | 03_機能一覧 |
| LCP | Largest Contentful Paint | 表示速度体感の指標 | 03_機能一覧 |
| INP | Interaction to Next Paint | 操作応答性の指標 | 03_機能一覧 |
| CLS | Cumulative Layout Shift | レイアウト安定性の指標 | 03_機能一覧 |
| アクセシビリティ | 多様な利用者が使える設計品質 | キーボード操作、コントラスト等 | 03_機能一覧 |
| 変更容易性設計 | 変更時の影響範囲を局所化し、拡張しやすくする設計方針 | FR-76としてP0採用 | 03_機能一覧, 04_画面遷移図 |
| 画面遷移図 | 画面間の移動関係を定義した設計資料 | SCR-IDと対応づける | 04_画面遷移図 |
| 301リダイレクト | 恒久的なURL移転を示すHTTPステータス | URL変更時の導線維持とSEO保護に利用 | 04_画面遷移図 |
| 受け入れ基準（AC） | 機能が完成したと判定するための具体条件 | Pass/Failの判定軸 | 05_受入基準 |
| Gherkin | Given/When/Thenで仕様を振る舞い記述する形式 | テスト観点の共通言語化に有効 | 05_受入基準 |
| 非機能要件（NFR） | セキュリティ・性能・可用性など機能以外の品質要件 | 機能要件と同等に受け入れ判定する | 05_受入基準 |
